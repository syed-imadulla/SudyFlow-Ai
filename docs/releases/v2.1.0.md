# Release Notes: v2.1.0

**Date:** July 21, 2026

## Overview
Phase 2.2 has been fully completed. This release marks the completion of the entire **Goal ↔ Planner synchronization foundation**, bridging the gap between high-level goals and day-to-day actionable planning.

## Completed Milestones

### Architecture & Synchronization
- **Goal ↔ Planner Synchronization**: Full two-way data consistency between Goal milestones and Planner time blocks.
- **Planner Synchronization Service**: Dedicated backend services (`PlannerService`, `GoalSyncService`) handle complex interdependent states cleanly.
- **Goal Lifecycle Cleanup**: Deleting goals or milestones correctly cascades into the planner without leaving orphaned blocks.
- **Application Rollback Strategy**: Implemented robust two-phase commits with rollback compensation in `GoalSyncService` to handle failures gracefully.

### UI & User Experience
- **Planner Scheduling Workflow**: A new, intuitive modal allows scheduling and updating milestones directly, featuring built-in conflict detection.
- **Workspace ↔ Planner Synchronization**: Real-time frontend updates ensure that when a milestone is scheduled, the UI updates instantly without requiring a hard refresh.
- **Dashboard Synchronization**: The Dashboard strictly reflects the actionable context with zero visual bloat.
- **UI Consistency Improvements**: Cleaned up the action bars across Workspace and Dashboard, removing large text buttons in favor of elegant minimalist icons, keeping tooltips informative.
- **Planner Scheduling UX**: Redesigned Planner block titles to emphasize the milestone name, demoting the Goal name to a clean, small metadata chip to prevent text wrapping and improve scanning.

### Engineering & Quality Assurance
- **Automated Testing & Integration Tests**: Built a comprehensive Jest test suite specifically targeting Goal ↔ Planner synchronization architecture.
- **API Error Standardization**: Replaced string-based errors with stable, machine-readable application error codes (e.g., `ERR_NOT_FOUND`, `ERR_VALIDATION`).
- **Structured Logging**: Replaced generic console logs with a centralized structured logger, significantly enhancing observability for production environments.

## Known Limitations
- No recurring planner events currently supported.
- Drag-and-drop rescheduling is not yet implemented.
- Focus mode is not fully integrated with the scheduling flow.
- AI scheduling enhancements are pending.

## Looking Ahead
The next milestone is **Phase 2.3**, which will focus on:
- Focus Mode
- Dashboard Intelligence
- Planner UX
- AI Productivity
